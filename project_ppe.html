PPE 착용 감지 모니터링 시스템

2025.08 · 개인 프로젝트 · YOLOv8 기반 안전모 / 조끼 착용 모니터링

PROJECT OVERVIEW

프로젝트 설명

작업 현장의 영상에서 사람과 PPE(안전모, 안전조끼)를 자동으로 감지하고, 인원 수 대비 PPE 완전 착용 비율을 실시간으로 계산·표시하는 모니터링 시스템이다.

YOLOv8 기반 객체 감지 모델을 사용해 Person / Hardhat / Safety Vest 클래스를 탐지하고, PPE 중심점이 사람 박스 안에 포함되는지를 기준으로 착용 여부를 판별한다. 프레임마다 총 인원 수, 완전 착용 인원 수, 착용률(%)을 HUD 형식으로 시각화하고, 일정 주기마다 평균 착용률을 CSV로 기록해 통계로 관리할 수 있도록 했다.

PROJECT GOAL

프로젝트 목적

  • CCTV / 작업 현장 영상을 기반으로 실시간 PPE 착용 여부 자동 판별
  • 총 인원 수 대비 PPE 완전 착용률을 직관적인 HUD 형태로 시각화
  • 일정 간격으로 착용률 통계를 CSV로 저장하여 추세 분석 가능하도록 구현
  • PPE 미착용 인원이 기준 이상 발생할 경우 경고 메시지로 즉시 피드백 제공
  • 향후 대시보드, 알림 시스템과 쉽게 연동할 수 있는 구조로 설계

  • 탐지 결과를 활용한 사람–PPE 매칭 로직(중심점 포함 여부 기반) 설계 및 구현

DEVELOPMENT PROCESS

단계별 진행 내용 · 역할 · 문제 해결

1단계. 문제 정의 & 모델 준비

  • 진행 내용 : “얼마나 많은 사람이 PPE를 제대로 착용하고 있는가?”를 숫자로 보여주는 것을 목표로 문제를 정의했다. YOLO 계열 모델 중 사용성이 높은 Ultralytics YOLOv8을 선택하고, best.pt 모델을 로딩해 Person / Hardhat / Safety Vest 클래스 탐지가 가능하도록 설정했다.
  • 담당 역할 : 모델 경로, 신뢰도 임계값(CONF_THRESHOLD) 등 핵심 하이퍼파라미터를 정하고, 테스트 영상(hardhat1.mp4)을 기반으로 모델이 실제 PPE 객체를 충분히 안정적으로 잡는지 검증했다.
  • 마주친 문제 : 신뢰도 임계값을 낮추면 잘못된 박스가 많아지고, 높이면 PPE 일부를 놓치는 문제가 발생했다.
  • 해결 방안 : 여러 신뢰도 값(0.25 ~ 0.6)을 테스트한 뒤, 0.4에서 “미검출 & 오검출” 간 균형이 가장 안정적인 것을 확인하고 기본값으로 채택했다.

2단계. 객체 감지 & PPE 매칭 로직

  • 진행 내용 : 한 프레임에서 사람, 안전모, 안전조끼를 각각 리스트로 분리하고, PPE 박스의 중심점이 사람 박스 내부에 들어가는지 확인해 해당 인원이 PPE를 착용한 것으로 판단하는 로직을 구현했다.
  • 담당 역할 : get_box_center(), center_inside()와 같은 유틸리티 함수를 설계해 코드를 모듈화하고, 인당 has_hat, has_vest를 체크한 뒤 둘 다 만족할 때만 완전 착용자로 카운트하는 방식을 정의했다.
  • 마주친 문제 : 카메라 각도와 사람 위치에 따라 PPE 박스 중심점이 사람 박스 경계 바깥으로 살짝 벗어나는 경우가 있었다.
  • 해결 방안 : 초기에는 IoU 기반 매칭도 고려했으나 구현 복잡도 대비 이점이 크지 않다고 판단했다. 대신 사람 박스가 너무 타이트하게 잡히지 않도록 영상 해상도와 리사이즈 방식을 조정해 중심점 기준 매칭이 안정적으로 동작하도록 보정했다.

3단계. 통계 집계 & CSV 저장

  • 진행 내용 : 프레임 단위로 계산한 총 인원 수와 완전 착용 인원 수를 리스트에 누적 저장했다가 일정 시간마다 평균값을 구해 CSV 파일로 저장하는 구조를 만들었다.
  • 담당 역할 : total_history, perfect_history를 활용해 구간 평균 인원 수와 평균 착용률을 계산하고, ppe_log.csv에 타임스탬프와 함께 기록하는 save_csv() 함수를 구현했다.
  • 마주친 문제 : YOLO 감지 특성상 특정 프레임에서만 일시적으로 인원을 놓치는 경우가 있어 단일 프레임 기준 통계는 들쭉날쭉했다.
  • 해결 방안 : 프레임 단위가 아닌 시간 구간 평균을 사용해 통계를 저장하도록 설계했다. 테스트 단계에서는 10초 간격으로, 실제 운영을 가정한 설계에서는 timedelta(hours=1) 기준으로 바꿔서 사용할 수 있도록 코드에 주석을 남겨두었다. PPE 착용률이 85% 이상이면 양호한 상태로 보는 기준도 함께 정리했다.

4단계. 시각화 & 경고 정책

  • 진행 내용 : PPE 착용 여부에 따라 사람 박스를 초록(완전 착용) / 빨강(미착용) 색상으로 구분해 그리고, 화면 좌측 상단에 Total / Perfect / Ratio 정보를 HUD처럼 표시했다. 미착용 인원이 2명 이상일 때는 No Wearing 경고 문구를 함께 띄우도록 했다.
  • 담당 역할 : 텍스트 위치(TEXT_POS)와 색상(LABEL_COLORS)을 설정값으로 분리해 Config 클래스로 관리하고, draw_text(), draw_box() 함수로 시각화 로직을 정리했다.
  • 마주친 문제 : 화면에 너무 많은 정보를 한 번에 표시하면 오히려 가독성이 떨어지는 문제가 있었다.
  • 해결 방안 : 핵심 지표만 남기고(총 인원, 완전 착용, 비율, 경고 문구), 색상과 텍스트 크기만으로도 상태가 빠르게 파악되도록 단순한 레이아웃으로 재구성했다. 향후 웹 대시보드와 연동할 때는 이 HUD를 그대로 스트리밍에 사용할 수 있도록 인터페이스를 유지했다.

OUTCOME & LEARNINGS

프로젝트 성과 및 얻은 인사이트

단순히 “PPE를 감지했다”에서 끝나는 것이 아니라, 얼마나 많은 사람이 규정을 지키고 있는지 수치로 보여주는 구조까지 구현했다는 점이 이 프로젝트의 핵심이다. 프레임 단위 탐지 결과를 시간 단위 통계로 집계하고, 평균 인원 수와 착용률을 CSV에 남겨 나중에 다시 분석할 수 있도록 만든 경험은, 현장에서 실제로 활용 가능한 컴퓨터 비전 시스템을 설계하는 감각을 길러줬다.

또한, 모델 성능만큼이나 판단 기준(중심점 포함 여부, 임계값, 경고 조건)을 어떻게 설계하느냐에 따라 시스템이 주는 인사이트가 달라진다는 점을 배웠다. 이 프로젝트를 통해, 추상적인 “딥러닝 모델”이 아니라 데이터 수집 → 모델 추론 → 로직 설계 → 통계/시각화까지 이어지는 전체 파이프라인을 설계·구현하는 데 자신감을 갖게 되었다.
