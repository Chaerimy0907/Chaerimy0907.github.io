화재 감지 자율주행 순찰 로봇

2025.09 – 2025.11 · 4인 팀 프로젝트 · ROS2 기반 자율주행 / 화재 감지 순찰 시스템

PROJECT OVERVIEW

프로젝트 개요

실내 환경에서 화재 위험을 조기에 감지하고, 로봇이 지정된 경로를 따라 자율적으로 순찰하며 예외 상황에도 안정적으로 대응할 수 있도록 설계한 프로젝트입니다. MQ-2, ENS160 센서를 통해 연기·가스 농도를 감지하고, ROS2 SLAM·Navigation2를 활용해 저장된 웨이포인트 기반 순찰을 수행합니다.

MY ROLE

담당 업무 및 기여

  • SLAM 기반 지도 생성 및 Navigation2 파이프라인 구성
  • 웨이포인트 기반 순찰 전략 설계 및 경로 구조 정의
  • Patrol Manager 노드 설계 및 구현(상태 기반 순찰 제어)
  • 정지 / 재개 / 복귀 등 예외 상황 제어 로직 구현
  • 센서 이벤트 및 배터리 상태를 순찰 제어와 연동하는 의사결정 로직 설계
  • VMware ↔ SBC 간 ROS2 통신 환경 구성 및 네트워크 이슈 디버깅

DEVELOPMENT PROCESS

개발 접근 방식

초기에는 FollowWaypoints 액션을 사용해 웨이포인트를 순서대로 실행하는 단순한 순찰 구조로 시작했습니다. 기본 주행은 문제없이 동작했지만, 순찰 중 정지·재개·복귀 명령과 배터리 조건, 화재 이벤트가 동시에 들어오는 실제 운영 시나리오에서는 제어 흐름이 쉽게 꼬이는 한계를 경험했습니다.

이를 해결하기 위해 순찰을 하나의 동작이 아니라 Idle / Patrolling / Paused / Returning과 같은 상태 전이(State Machine) 관점에서 다시 설계했습니다. 모든 외부 명령을 상태 변경으로 해석하고, 실제 주행은 단일 흐름에서만 실행되도록 구조를 단순화하여 예외 상황에서도 예측 가능한 동작을 유지할 수 있도록 개선했습니다.

TECH STACK

사용 기술

PROBLEM SOLVING

문제 해결 사례

  • 순찰 중 정지 명령이 일시적으로만 적용되던 문제를, 액션 cancel + 상태 기반 제어로 개선하여 항상 즉시 정지되도록 구조 변경
  • 반복 순찰 과정에서 로봇 위치가 튀는 현상을 TF 타임스탬프·배터리 조건 충돌 문제로 분석하고, 배터리 로직과 순찰 로직을 분리해 안정적으로 동작하도록 수정
  • RViz에서는 맵이 보이지만 외부 시각화에서는 표시되지 않던 이슈를 /map 토픽의 QoS 정책을 통일하여 해결
  • VMware ↔ SBC 간 ROS2 통신이 간헐적으로 끊기는 문제를, 네트워크/데몬 상태 확인 및 재시작(daemon stop/start)으로 런타임 환경 문제로 구분해 진단

OUTCOME & LEARNINGS

성과 및 배운 점

20회 이상의 테스트에서 순찰 성공률 95%, 화재 이벤트 발생 시 정지 명령 반응 시간 평균 0.28초 수준으로 동작을 안정화했습니다. 단순히 로봇을 움직이는 수준이 아니라, 실제 운영을 고려한 제어 구조를 설계·구현한 경험입니다.

이 프로젝트를 통해 자율주행에서는 알고리즘 자체보다 제어 흐름과 상태 관리 설계가 더 중요하다는 점을 체감했고, 분산 환경(ROS2, VMware, SBC)에서는 코드 못지않게 런타임 환경과 네트워크 상태를 함께 보는 시야가 필요하다는 것을 배웠습니다.
